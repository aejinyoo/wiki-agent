# docs/

Cowork 기획 ↔ Claude Code 구현 사이에 컨텍스트를 이어주는 작업 로그.

## 구조

```
docs/
├── current.md              ← 새 세션 시작할 때 첫 진입점
├── features/
│   ├── _template.md        ← 새 작업 시작 시 복사
│   └── <feature>.md        ← 기능당 파일 1개
└── archive/                ← 완료된 과거 spec
```

## 워크플로우

**새 작업 시작 (Cowork)**
> "`features/_template.md`를 `features/<기능명>.md`로 복사하고 지금까지 논의한 내용을 요약·다음 액션에 정리. `current.md`에도 등록."

**구현 시작 (Claude Code)**
> "`features/<기능명>.md` 읽고 이어서 진행. 진행/다음 액션 업데이트."

**세션 종료 시**
> "한 일·다음 할 일·막힌 점을 `features/<기능명>.md`에 업데이트."

**완료 시**
- 상태를 `완료`로 변경 → `current.md`에서 제거 → 필요하면 `archive/`로 이동

## 파일 구성 (기능당 1개 파일)

한 파일 안에 섹션으로 구분:
- **요약** — 무엇을/왜 + 스코프
- **진행** — 날짜별 로그 (최신순)
- **다음** — 체크박스
- **결정** — 큰 방향 전환만 (옵션)
- **링크** — 관련 파일/커밋/외부 (옵션)
